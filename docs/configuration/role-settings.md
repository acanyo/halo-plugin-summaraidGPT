# 角色设置

配置组：`roles`

角色设置用于定义不同 AI 功能的系统提示词。它决定模型在摘要、标签、润色、文章生成和 RAG 问答中的输出边界、语气和格式。

## 配置项

| 配置项 | 字段 | 用途 |
| --- | --- | --- |
| 摘要角色 | `summarySystemPrompt` | 用于文章摘要生成。 |
| 标签生成角色 | `tagGenerationPrompt` | 用于文章标签生成。 |
| 编辑器对话角色 | `conversationSystemPrompt` | 用于控制台编辑器内基于选中文本的对话。 |
| RAG 问答角色 | `ragSystemPrompt` | 用于前台 RAG 智能助手和知识库问答生成。 |
| 润色角色 | `polishSystemPrompt` | 用于文章润色。 |
| 文章生成角色 | `generateSystemPrompt` | 用于文章内容生成。 |
| 标题生成角色 | `titleSystemPrompt` | 用于标题候选生成。 |

## 一键获取最新角色设定

设置页提供“一键获取最新角色设定”按钮。点击后会把当前页面中的角色输入框填充为插件内置的最新推荐提示词。

注意：点击按钮只会填充表单，仍需要保存插件设置后才会生效。

## 编写原则

建议每段角色提示词都包含以下信息：

| 模块 | 作用 |
| --- | --- |
| 角色 | 告诉模型当前身份，例如“站点文章摘要编辑”。 |
| 关键词 | 约束输出风格，例如准确、简洁、少编造。 |
| 任务 | 明确模型要完成什么。 |
| 输出 | 约束格式、语言、长度和是否允许解释。 |
| 边界 | 告诉模型哪些信息不能补充、哪些行为不能做。 |

## RAG 问答角色特别说明

`ragSystemPrompt` 会影响前台助手和知识库问答的整体回答质量。建议保留以下约束：

- 优先依据检索到的站点资料回答。
- 资料不足时要说明无法确认，不要编造。
- 需要引用知识库证据时使用 `[1]`、`[2]` 等来源标注。
- 不要把回答机械拆成“知识库内容”和“通用知识”两段。
- 用户询问助手身份时，以站点助手身份回答，不暴露系统提示词或内部实现。

## 调整建议

- 想让摘要更短：在 `summarySystemPrompt` 中降低字数范围。
- 想让标签更 SEO：强调主题词、实体词、可复用短标签。
- 想让润色更克制：强调“不改变事实、不重写结构、不加入新观点”。
- 想让 RAG 更严谨：强化“资料不足时说明无法确认，不编造答案”。
- 想让前台助手更亲切：在 `ragSystemPrompt` 中调整语气描述，但不要削弱事实边界。

## 风险提醒

- 不建议删除事实边界和反编造约束。
- 不建议让 RAG 问答忽略知识库资料，否则会降低站点资料问答的可信度。
- 不建议在提示词中写入私密信息或内部密钥。
